# 当前为Obsidian开发插件的初始化项目，帮我开发一个Obsidian插件，插件的作用是一个闪卡相关的产品，可以帮助我学习和复习知识点：

- 可以自定义标签，自动扫描标签页面的单词查分成卡片
- 每个文件只识别第一个tag即可，每个文件对应着一个题库，讲文件中的每个卡片都添加到这个题库中
- 导入格式要求
    1.  使用“---div---”进行问题和答案的分割
    2.  使用 “<->”进行卡片的分割
    3.  问题和答案都支持markdown格式

    ```markdown
    #wordTag

    问题写在这个地方
    ---div---
    答案写在这个地方
    <->

    问题写在这个地方
    ---div---
    答案写在这个地方
    <->
    ```

- 每个题库如果新题都学习完了，那么之后在点开就是复习
- 插件的首页显示设置按钮和题库List
    - 题库List展示题库名称，学习次数
- 使用FSRS6算法
- 可以设置“每日复习数量”和“每日复习数量”
- 可以设置按顺序学习和乱序学习
- 卡片界面和操作要求
    - 卡片header部分展示当前题库的名称和答题进度，答题进度显示“当前的第几题/总题数”，header最后边展示当前学习时间，学习时间后面是关闭按钮
    - 内容区上面显示题目，下面是答案展示区域
        - 初始卡片展示内容部分只展示问题，用户点击“显示答案”按钮后展示答案
    - 底部的footer部分显示操作按钮：
        - 初始卡片footer部分展示按钮“显示答案”
        - 用户点击“显示答案”按钮后分别展示“重来（1分钟）” 、“困难（1天）”、“良好（3天）”、“简单（10天）”、“辣鸡（21天）”、“上一题”，”重来“对应本次练习中还会根据相应算法出现此单词，“困难”代表一天后的练习还会出现此单词，“良好”代表3天后练习还会出现此单词，“简单”代表十天后练习还会出现此单词，“辣鸡”代表21天后的练习还会出现此单词，”上一题“代表重新做一遍上一次出现的题目。
    - 在电脑端的快捷键：
        - 空格键映射“显示答案”按钮
        - 数字1键映射“重来”
        - 数字2键映射“困难”
        - 空格键和数字3键映射“良好”
        - 数字4键映射“简单”
        - 数字5键映射“辣鸡”
        - 数字6键映射“上一题”
    - 如果有我没有提到的地方可以参考其他闪卡类产品，或者你帮我出谋划策

# 技术要求：

- 前端框架使用最新版本的react
- 需要其他的库可以自行安装
- 交互上要有平滑的动画过度
- Obsidian插件开发的官方文档可供参考：https://docs.obsidian.md/Plugins/Getting+started/Build+a+plugin

# 为我的插件添加新功能-刷题模式：

- 刷题模式是一个独立的模式，用户可以选择进入刷题模式
- 刷题模式下，用户可以选择一个题库进行刷题
- 刷题模式下，用户可以选择题目的数量，比如10题，20题，30题等
- 刷题模式下，题目顺序为随机顺序
- 刷题模式下，用户答完所有题目后，展示一个总结页面，展示用户的正确率，错误题目List等信息
- 刷题模式下，用户可以选择重新开始刷题，或者返回主页面
